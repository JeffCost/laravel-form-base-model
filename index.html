    Why model forms?

    Forms have their own data needs that are separate from the underlying data-types that they may be used to modify. Forms also have different validation needs than these underlying data-types.

    You may need to populate a drop-down select field with static values from an array or from methods which fetch data from other models. There's no reason to prepare, populate and validate forms in the controller.

    Forms may also be constructed "wizard style" which splits up a large form into many smaller forms which span many pages. By handling this with the form model your controller won't be filled with session data management code.

    In short, they provide a place to put things that have traditionally belonged in neither the controller nor our database models.

    A video explaining a bit more about form models can be found here.

    Features

    • Handles form validation
    • Easy form field re-population
    • Provides a location for defining custom validation rules
    • Easily manage form field data across multiple page forms

    Bundle Installation

    Install with artisan

    php artisan bundle:install form-base-model

    or, clone the project into bundles/form-base-model.

    Then, update your bundles.php to auto-start the bundle.

    return array(
    'form-base-model' => array( 'auto' => true ),
);

    Notice: The form-base-model bundle comes with examples that will auto-route to http://yoursite/form_examples. Remember to remove this from bundles/form-base-model/start.php when you're finished with the examples.
